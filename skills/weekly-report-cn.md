# 周报生成器

此技能帮助您创建专业的周报，包含您的成就、任务和计划。生成的报告会根据您的组织格式偏好进行定制，可以针对不同的受众(团队更新、管理层报告等)进行自定义。

## 基本用法

```
/weekly-report-cn
```

我会引导您创建周报，询问以下信息:
- 时间段（例如，"本周"，"2026年4月17-23日"）
- 您偏好的报告格式
- 目标受众
- 您的主要成就
- 任何挑战或阻碍
- 您下周的计划

## 高级用法

您可以预先提供部分或全部信息:

```
/weekly-report-cn 本周我完成了前端重新设计，修复了3个关键bug，并主持了团队回顾会。下周我将专注于API集成。这份报告是给我的经理的。
```

## 报告格式

此技能支持多种标准报告格式:

1. **标准格式**
   - 成就
   - 进行中的工作
   - 阻碍/挑战
   - 下周计划

2. **STAR方法**
   - 情境（具体情况）
   - 任务（需要完成的工作）
   - 行动（采取的措施）
   - 结果（取得的成果）

3. **管理层摘要**
   - 执行概述
   - 关键指标
   - 团队亮点
   - 资源需求

4. **自定义格式**
   - 您可以描述您偏好的结构

## 有效报告的技巧

- 尽可能具体描述成就，包含可衡量的结果
- 保持条目简洁明了
- 强调影响而非仅仅是活动
- 对于管理层报告，强调业务价值
- 对于团队报告，包含更多技术细节
- 在可能的情况下包含相关指标（例如，"减少了40%的加载时间"）

## 示例

### 团队更新示例

```
/weekly-report-cn 本周我完成了用户认证模块的重构，参与了3次代码审查，并修复了生产数据库连接问题。由于DevOps管道问题，我在部署到测试环境时受阻。下周我将专注于新的报告功能。这是给我的工程团队的。
```

### 管理层报告示例

```
/weekly-report-cn 时间段: 2026年4月17-23日。完成了第二季度客户满意度分析，显示12%的改进；培训了2名新团队成员使用数据管道；提前交付了执行仪表板。面临欧洲地区数据质量问题的挑战。计划下周开始预算预测。格式：面向总监级别的管理层摘要。
```
