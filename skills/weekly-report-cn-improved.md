# 周报生成器

这是一个专为中文用户设计的周报/工作总结生成工具。无论您是需要向团队汇报工作进展，向管理层提交成果总结，还是向导师提交研究进展，此技能都能帮助您快速创建专业、结构化的报告。

## 报告类型

此技能支持多种报告类型：

- **工作周报** - 适合职场环境，总结工作成果与计划
- **项目进展** - 专注于特定项目的状态、里程碑和风险
- **研究记录** - 适合学术或研究场景，记录实验、阅读和思考
- **学习总结** - 适合学生或培训环境，总结知识获取与应用
- **日报/双周报/月报** - 支持不同时间跨度的汇报需求

## 基本用法

直接调用技能，我会引导您完成报告创建：

```
/weekly-report-cn
```

## 高级用法

您可以在调用时直接提供部分或全部信息，加快报告生成：

```
/weekly-report-cn 本周完成了数据库优化工作，提高查询速度30%；参与了产品规划会议；解决了3个核心bug。下周计划开始用户界面重构。报告对象是我的技术主管。
```

或者指定特定格式和类型：

```
/weekly-report-cn 类型:研究记录 时间:4月18-24日 内容:完成了实验数据收集和对比分析 格式:STAR方法
```

## 信息收集指南

为创建高质量报告，我会询问以下关键信息（您也可以直接在命令中提供）：

1. **报告类型** - 工作周报、项目进展、研究记录等
2. **时间范围** - 报告覆盖的时间段（如"本周"、"4月17-23日"）
3. **受众对象** - 面向谁（团队、经理、导师、客户等）
4. **完成内容** - 已完成的工作、任务或成就
5. **进行中工作** - 正在进行但尚未完成的事项
6. **遇到的挑战** - 问题、障碍或需要支持的方面
7. **后续计划** - 下一阶段的工作重点或目标
8. **格式偏好** - 您想要的报告结构类型

## 报告格式选项

根据需要选择最合适的格式：

1. **标准格式**（默认）
   - 已完成工作
   - 进行中工作
   - 遇到的挑战/阻碍
   - 下周/下阶段计划

2. **STAR方法**（适合成就导向的报告）
   - 情境（Situation）- 面临的背景或问题
   - 任务（Task）- 需要完成的具体工作
   - 行动（Action）- 采取的具体措施
   - 结果（Result）- 取得的成果与影响

3. **管理层摘要**（适合向高层汇报）
   - 执行概述（总体进展与状态）
   - 关键指标（数据与成果）
   - 团队/项目亮点（特别成就）
   - 风险与资源需求（需要关注的问题）

4. **研究型报告**（适合学术或研究场景）
   - 研究目标回顾
   - 本周进展（实验/阅读/分析）
   - 发现与思考
   - 下阶段研究计划

5. **项目状态报告**（适合项目管理）
   - 项目概览与健康度
   - 已完成里程碑
   - 风险与问题
   - 下阶段计划与预期

6. **自定义格式**
   - 根据您的具体需求定制结构

## 撰写有效报告的技巧

### 针对不同受众的建议

**向团队/同事汇报时**：
- 包含足够的技术细节以便理解
- 明确标注需要团队协助的地方
- 提及可能影响其他人工作的变更

**向管理层汇报时**：
- 强调业务价值和结果影响
- 使用数据和指标量化成果
- 简化技术细节，突出战略意义
- 清晰标识需要决策支持的事项

**向导师/学术环境汇报时**：
- 关注研究方法和理论框架
- 详述发现的意义和局限性
- 提出下一步研究方向的思考

### 通用撰写建议

- **成果导向**：强调工作产生的影响，而非仅列出活动
- **量化成果**：尽可能用数据说话（"提高了30%"、"减少了25分钟"）
- **问题解决**：不只指出问题，也提出解决方案或需要的支持
- **优先级**：重要事项放在前面，次要细节靠后
- **视觉清晰**：使用项目符号、分段和小标题提高可读性
- **简明扼要**：保持内容精炼，避免不必要的细节

## 示例

### 工作周报（团队版本）

```
/weekly-report-cn 本周（4月18-24日）我完成了用户认证模块重构，参与了3次代码审查，修复了2个生产环境bug。由于API文档不完整，集成测试进度受阻。下周计划完成支付接口集成和单元测试编写。这份报告给我的开发团队。
```

### 项目进展报告（管理层版本）

```
/weekly-report-cn 类型:项目进展 时间:本月 受众:部门经理 内容:电商平台改版项目已完成70%，提前一周；用户测试反馈积极，满意度达85%；已节省预算8%。面临挑战:移动端兼容性问题导致测试周期延长。下阶段:准备5月15日正式发布，需增加一名UI设计师支持。格式:管理层摘要
```

### 研究进展（学术版本）

```
/weekly-report-cn 类型:研究记录 时间:4月第三周 受众:导师 内容:完成了数据集预处理和初步模型训练；发现原有算法在大规模数据上准确率下降15%；阅读了5篇相关论文并整理了笔记。下周计划设计改进算法并开始撰写论文方法部分。格式:研究型报告
```

## 日期格式和语言偏好

报告默认使用中文日期格式（如"2026年4月24日"而非"April 24, 2026"），如有其他偏好请说明。

## 保存和分享

生成的报告采用Markdown格式，便于：
- 复制到邮件、即时通讯工具或文档系统
- 转换为其他格式（如Word、PDF）
- 保存为文件供日后参考

如需将报告保存为文件，可在生成后说明，我会提供保存建议。
